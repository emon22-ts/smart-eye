System Overview

Multi-Modal Ocular Health Screening

Three independent data streams — CNN image classifier, real-time webcam fatigue monitor, symptom questionnaire — fused through a Mamdani fuzzy engine into a single explainable Ocular Health Index.

Input 1 — Vision
🖼 Fundus / Anterior Image
EfficientNetB0 classifies into 4 disease categories. Grad-CAM generates an explainability heatmap over the winning class region.
EfficientNetB0Grad-CAM4-class CNN~80 ms
Input 2 — Fatigue
👁 Live Webcam Monitor
68-point facial landmarks compute Eye Aspect Ratio (EAR) at ~8 Hz. Detects drowsiness and blink compliance. No webcam frames leave the browser.
dlib 68-ptEAR tracking~8 Hz~0.03 ms

📋 Symptom Questionnaire
Structured Likert-scale survey (1–5) captures subjective burden: blurred vision, pain, photophobia, redness, floaters, and more.

Decoupled Mamdani System

_RELIABILITY_
RECOMMEND_RECAPTURE
Clinical Safety Guarantee: Urgency is derived exclusively from PRI. A high-confidence pathology finding triggers IMMEDIATE_CLINICAL_PRIORITY even at eye_fatigue=100. This is structural — not a rule patch. Fatigue affects only the SRI advisory caveat.
Verified by automated test: test_high_pathology_triggers_urgency_regardless_of_fatigue parametrises eye_fatigue across [0, 50, 98, 100] — all yield IMMEDIATE_CLINICAL_PRIORITY when cnn_confidence = 0.92.
Membership Functions
Triangular MFs for mid-range (Low, Medium, High). Trapezoidal shoulder MFs at universe boundaries guarantee full coverage — no input can fall outside all MFs and produce NaN.
Rule Structure
Expert-defined 3×3 rule blocks (9 rules per system). Fuzzy AND = minimum t-norm. Example: IF confidence is High AND symptoms is Moderate THEN PRI is High.
Defuzzification
Centroid method (centre of gravity). If aggregated output is non-finite, _safe_simulate() catches it and returns neutral fallback = 50 — no unhandled exception propagates to the API.
